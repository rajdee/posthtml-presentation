Преимущество постпроцессоров:

  1. Возможность самостоятельного расширения
  2. Гибкость и недетерминированность
  3. Стандартный синтаксис
  4. Высокая производительность

История развития постпроцессоров

  1. 2012 - Rework (TJ Holowaychuk)
  2. 2013 - PostCSS (Andrey Sitnik)
  3. 2015 - PostHTML ( The Protein Corp. )

Экосистема PostHTML

  • PostHTML Core ( posthtml-parser, posthtml-render )
  • Plugins ( PostHTML Plugins Catalog )
  • Bindings ( posthtml-cli, gulp-posthtml, grunt-posthtml, posthtml-loader, )
  • Middlewares ( Koa, Hapi, Express, Electron )
  • PostHTML Gitter

    <div elem="nose" mods="size:long">Nose</div>
</div>

posthtml-bem (mix-example)

      <div
  block="animal"
  mix="block:elephant elem:trunk mods:[size:short broken],
  block:cow mods:[moo]">
      <div elem="nose" mods="size:long">Nose</div>
</div>

          $ html2react index.html -c es5 -m es6 -e js -o ./components

header.js

      import React, { Component } from 'react';
import Heading from './heading';

class Header extends Component {
  render() {
    return (
      <header className="header"></header>
        <Heading></Heading>
      </header>
    );
  }
}
export default Header;

heading.js

      import React, { Component } from 'react';
  class Heading extends Component {
    render() {
      return <h1 className="heading"<Hello, world!</h1>;
    }
  }
  export default Heading;
